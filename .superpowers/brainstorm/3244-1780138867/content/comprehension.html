Comprehension Check Interaction

How should users interact with the pipeline-generated questions and answers?

A

Reveal-on-Demand List (Recommended)

List all questions in the Q&A tab. Click "Show Answer" to reveal the pipeline's answer one at a time. Track which ones have been revealed.

Pros

  • Simple, no state machine needed
  • User controls pace — skip, reveal, re-hide
  • Low implementation effort
  • Works for casual browsing and study

Cons

  • Less engaging than quiz mode
B

Quiz Mode (One at a Time)

Show one question at a time with a text input. User types their answer, then clicks "Check" to compare with the pipeline's answer + self-rate (knew it / partly / didn't know).

Pros

  • Genuinely interactive
  • Self-assessment creates study value
  • Progress tracking possible

Cons

  • More complex state (current question, score, progress)
  • Input + comparison + rating = more components
  • Text answers = hard to auto-grade (needs human self-rate)
C

Side-by-Side Comparison

Split view: left side shows all questions, right side shows all answers. Toggle to hide/show answers globally. Simple reference card.

Pros

  • Quick overview of all Q&A pairs
  • Good for casual reference
  • Trivially simple to implement

Cons

  • Not really "interactive"
  • No study/retention value
