Graph Rendering Library

This choice affects visual graph rendering (item 2), pan/zoom (item 4), and overall complexity

A

@xyflow/react + Dagre Layout (Recommended)

React-first graph library. Built-in pan, zoom, minimap, node/edge rendering. Add @dagrejs/dagre for automatic hierarchical layout.

Pros

  • Built-in pan + scroll zoom (kills item 4)
  • Polished nodes, edges, handles
  • React component API — matches codebase style
  • Custom node types for document/concept/argument/question
  • Minimap and controls components included

Cons

  • 2 dependencies (~150KB gzipped total)
  • Dagre is hierarchical — not force-directed
  • Less flexible than raw d3 for unusual layouts
B

d3-force + Canvas

Use d3-force for force-directed layout and raw canvas/SVG for rendering. Full control.

Pros

  • Force-directed layout looks great for knowledge graphs
  • Impresses users (bouncy nodes)
  • Maximum flexibility

Cons

  • Must implement pan, zoom, node rendering, edge rendering manually (doubles the work)
  • Imperative API, fights React's declarative model
  • More code, more bugs, more tests
  • D3 is ~250KB alone
C

@xyflow/react + Optional d3-force Layout

Use xyflow for rendering + interactions, but swap Dagre for d3-force as the layout engine. Best of both worlds.

Pros

  • All of A's React integration + pan/zoom
  • Force layout with xyflow nodes
  • Can start with Dagre, add d3-force later

Cons

  • 3 dependencies (xyflow + d3-force + dagre for fallback)
  • Slightly more complex
