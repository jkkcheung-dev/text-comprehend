CSS Approach for Dashboard

How to make the dashboard look polished with minimal friction

A

CSS Modules (Recommended)

Zero config — Vite handles *.module.css natively. Scoped class names prevent collisions. Co-located with components. Use CSS custom properties for theming.

Pros

  • No new dependencies
  • Scoped by default
  • Co-located with components
  • Full CSS flexibility (grid, animations, custom props)

Cons

  • Must write raw CSS
  • No design tokens out of the box
B

Tailwind CSS

Utility-first framework. Fastest prototyping. Large ecosystem. Requires @tailwindcss/vite plugin + config.

Pros

  • Rapid prototyping
  • Design tokens built-in (spacing, colors, typography)
  • Huge ecosystem

Cons

  • Adds 2-3 dependencies
  • Long inline class strings in JSX
  • Paradigm shift from CSS
  • v4 breaking changes from v3
C

CSS Modules + Design Tokens

Option A plus a lightweight tokens file (tokens.module.css) with CSS custom properties for colors, spacing, typography, shadows. No extra deps.

Pros

  • All of A's benefits
  • Consistent design language
  • Easy dark mode via custom props
  • Zero extra dependencies

Cons

  • Slightly more upfront setup
